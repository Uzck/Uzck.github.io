定义注解

上面的代码就是一个最简单的注解类,定义注解的时候需要元注解(meta-annotation)。

@Target用来定义注解使用的地方(方法或者域等)

@Retention用来定义注解在哪一个级别可用(源代码、类文件或者运行时)

标准注解

  1. @Override: 表示当前方法定义将覆盖超类中的方法
  2. @Deprecated: 表示方法在未来版本将被废弃
  3. @SuppressWarnings: 关闭不当的编译器警告

元注解

  1. @Target: 表示注解可以使用的地方，可能的参数包括
    1. CONSTRUCTOR:构造器声明
    2. FIELD: 域声明
    3. LOCAL_VARIABLE: 局部变量声明
    4. METHOD: 方法声明
    5. PACKAGE: 包声明
    6. PARAMETER: 参数声明
    7. TYPE: 类、接口或enum声明
  2. @Retention: 表示需要在什么级别保存该注解信息，可选的参数包括
    1. SOURCE: 注解将被编译器丢弃
    2. CLASS: 注解在class文件可用，但会被VM丢弃
    3. RUNTIME: VM将在运行期也保留注解，可以通过反射机制读取该注解的信息
  3. @Documented: 将此注解包含在Javadoc中
  4. @Inherited: 允许子类继承父类中的注解

注解元素

注解元素可用类型

上面UseCase里的id和description称为注解元素，注解元素的所有可用类型:

  1. 所有基本类型
  2. String
  3. Class
  4. enum
  5. Annotation
  6. 以上类型的数组

使用其他类型时编译器会报错

####　默认值

所有的注解元素都要有确定的值，因此在没有提供值的时候必须设置默认值。非基本类型的元素不能以null作为默认值

Spring MVC @DateTimeFormat失效

发表于 2018-11-17 | 分类于 Spring

Spring MVC有自己处理表单的库，在使用时，需要在相应的JSP文件头部加上

<%@ taglib prefix="mvc" uri="http://www.springframework.org/tags/form" %>，如果通过maven的create from archetype时，还需要加上<%@ page isELIgnored="false"来避免使用<mvc:select items/>失效。

如果表单里有时间，可以在对应的实体类的time属性上加@DateTimeFormat(pattern="")注解，如@DateTimeFormat(pattern="yyyy-MM-dd")，这样在填表单数据时，只需要按1995-03-20的格式，服务端就能够正确的解析数据。

本来应该成功的，结果报了无法解析的错误，环境spring5.1，在网上查阅相关博客之后，有两点：

  1. 在springmvc-servlet.xml添加<mvc:annotation-driven/>
  2. 添加joda-time的包，版本大于2.5(验证后发现不需要)

添加了<mvc:annotation-driven/>后能正常解析数据了。
